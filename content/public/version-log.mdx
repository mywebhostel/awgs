## Versioning Policy Notes

- AWGS standard versions should not overwrite prior public standard versions without an archive path.
- Normative changes should be made in the canonical standard source before presentation data is updated.
- Informative guidance, knowledge base articles, and assistant behavior should cite the applicable AWGS version.
- Future certification or assurance programs should be versioned separately from conformance claims unless the standard defines otherwise.
