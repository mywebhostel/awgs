# AWGS Change Log

This public change log records changes to the AWGS standard and supporting guidelines. It does not record website implementation, hosting, build, or repository tooling changes.

## 2026-06-07

### AWGS 1.0 Foundation Specification

- Published the AWGS 1.0 Foundation Specification as a Public Draft Standard.
- Established the AWGS purpose and vision for trustworthy interactions between autonomous agents and digital services.
- Defined vendor neutrality requirements for AWGS compliance.
- Defined the AWGS 1.0 scope, including in-scope and out-of-scope system categories.
- Added eight core principles: discoverability, interoperability, transparency, accountability, trust, privacy, safety, and human oversight.
- Added nine governance domains: discoverability, interoperability, identity, authorization, transparency, accountability, trust, privacy, and safety.
- Defined three compliance levels: Level A, Level AA, and Level AAA.
- Added conformance claim expectations and examples.
- Added the assessment model covering automated, semi-automated, and manual assessment methods.
- Added governance requirements for identity, authorization, agent access, privacy, risk, trust, incident handling, audit, and policy management.
- Added standard governance, versioning, normative content, informative content, glossary, future directions, and conclusion sections.

### Guidelines and Supporting Guidance

- Added implementation guidance organized around AWGS principles, governance domains, conformance claims, and assessment evidence.
- Added planned knowledge base topics for agent-ready services, evidence collection, human oversight workflows, and agent identity with delegated authority.
- Identified future AWGS directions including agent-to-agent governance, multi-agent systems, autonomous transaction governance, identity frameworks, federated trust networks, reputation systems, continuous compliance, and safety certification.
