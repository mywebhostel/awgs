Agentic Accessibility Initiative

Building the foundation for a trusted Agentic Web

The Agentic Accessibility Initiative promotes open, vendor-neutral principles and standards for trustworthy interaction between autonomous agents and digital services.

Explore AWGS 1.0Read the Vision
What is Agentic Accessibility?

Access through trusted digital representatives

The principle that individuals and organizations should be able to interact with digital systems through trusted autonomous agents acting on their behalf.

Authorized delegation

Agentic access should let people and organizations delegate digital interaction to trusted representatives without forcing every service interaction to remain manual.

Standards-based trust

Agentic services need shared expectations for discovery, authority, transparency, accountability, privacy, safety, and oversight.

Governance Need

Why the Agentic Web needs governance

As agents discover services, request access, exchange data, and perform actions, digital systems need shared expectations for trust, authorization, transparency, accountability, privacy, safety, and oversight.

First Standard

AWGS as the first standard

AWGS turns the broader Agentic Accessibility vision into concrete governance domains, compliance levels, conformance expectations, and assessment guidance for agentic digital services.

Read AWGS overview
Public Draft

Neutral, implementation-independent guidance

AWGS does not require a specific vendor, product, framework, protocol, or technology. It defines governance expectations that can be implemented across different operational environments.

Core Principles

Core initiative principles

These principles guide Agentic Accessibility vision work and help frame the standards that support trusted agent-mediated interaction.

Delegated Access

People and organizations should be able to authorize trusted agents to interact with digital services on their behalf.

Machine-Readable Participation

Digital services should expose capabilities, policies, and constraints in forms that autonomous agents can discover and understand.

Transparent Representation

Agent actions should be understandable to users, services, and reviewers, especially when outcomes have significant consequences.

Accountable Automation

Agent-mediated interactions should preserve auditability, provenance, oversight, and meaningful routes for correction or escalation.

Compliance Model

Level A, AA, and AAA at a glance

AWGS 1.0 uses progressive compliance levels to describe readiness, interoperability, and governance maturity. Until AWGS 1.1 requirements and success criteria exist, Level A, Level AA, and Level AAA should be treated as maturity targets, not independently certified claims.

Level AAgent Awareness

Agent Ready

The system provides sufficient information for agents to discover and understand available capabilities.

  • Capability visibility
  • Machine-readable documentation
  • Basic governance readiness
Level AAAgent Interaction

Agent Interoperable

The system supports secure and governed interaction between agents and digital services.

  • Secure interaction
  • Delegated permissions
  • Governed service access
Level AAAAgent Trustworthiness

Agent Governed

The system demonstrates mature governance, trust, transparency, and safety controls.

  • Mature assurance
  • Traceability
  • Trust, safety, and oversight controls
